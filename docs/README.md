# Documentação do Projeto

Esta pasta contém toda a documentação relacionada ao sistema, incluindo modelagem, requisitos e diagramas.

O objetivo é fornecer uma visão completa do projeto, facilitando o entendimento da aplicação tanto em nível técnico quanto funcional.

---

## Estrutura

### diagrams/
Contém os diagramas do sistema:

- Diagramas de sequência  
- Diagrama de casos de uso  
- Diagrama de classes  

---

### architecture.md
Descreve a arquitetura geral do sistema, incluindo os componentes principais e o fluxo de dados.

---

### persona.md
Define o perfil do usuário alvo do sistema, incluindo objetivos e dores.

---

### user-stories.md
Lista as histórias de usuário que descrevem as funcionalidades do sistema sob a perspectiva do usuário.

---

### use-cases.md
Apresenta os casos de uso detalhados do sistema, baseados nas histórias de usuário.

---

### requirements.md
Contém os requisitos do sistema:

- Requisitos funcionais (RF)  
- Requisitos não funcionais (RNF)  

---

## Objetivo

A documentação tem como objetivo:

- Organizar o desenvolvimento do sistema  
- Facilitar manutenção e evolução  
- Melhorar a compreensão do projeto  
- Servir como apoio para apresentação e avaliação  

---

## Observações

Os documentos podem ser atualizados conforme novas funcionalidades sejam adicionadas ao sistema.
